How to get an instance of a page when you navigate to it in UWP

26 May 2019

By default when navigating a frame to a page, UWP doesn’t give you back a page instance. So the only way of passing parameters is to use the parameter parameter. But having an instance to the page can have many other benefits: For example it allows the child page to have custom events and the parent page can subscribe to them.

However Frame has an event called Navigated that can help us to write an extension method to get a reference to the actual instance:

 1using Windows.UI.Xaml.Media.Animation;
 2using Windows.UI.Xaml.Navigation;
 3
 4namespace Windows.UI.Xaml.Controls
 5{
 6    public static class FrameExtensions
 7    {
 8        /// <summary>
 9        /// Navigates to a page and returns the instance of the page if it succeeded,
10        /// otherwise returns null.
11        /// </summary>
12        /// <typeparam name="TPage"></typeparam>
13        /// <param name="frame"></param>
14        /// <param name="transitionInfo">The navigation transition.
15        /// Example: <see cref="DrillInNavigationTransitionInfo"/> or
16        /// <see cref="SlideNavigationTransitionInfo"/></param>
17        /// <returns></returns>
18        public static TPage Navigate<TPage>(
19            this Frame frame,
20            NavigationTransitionInfo transitionInfo = null)
21            where TPage : Page
22        {
23            TPage view = null;
24            void OnNavigated(object s, NavigationEventArgs args)
25            {
26                frame.Navigated -= OnNavigated;
27                view = args.Content as TPage;
28            }
29
30            frame.Navigated += OnNavigated;
31
32            frame.Navigate(typeof(TPage), null, transitionInfo);
33            return view;
34        }
35    }
36}

Suppose we have page called Page1:

 1public sealed partial class Page1 : Page
 2{
 3    public Page1()
 4    {
 5        this.InitializeComponent();
 6    }
 7
 8    public void Init(string param1, int param2)
 9    {
10        // Initialize the page
11    }
12}

You can navigate to it like so:

1Page1 page = frame.Navigate<Page1>();
2// You can now do stuff with the page
3// for example pass in as many parameters as you want
4page.Init("Hello World", 42);

Note: Page1.Init will be called after Page1.OnNavigatedTo and before Page1.Loading.

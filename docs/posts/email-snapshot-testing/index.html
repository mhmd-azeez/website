Testing Email Templates in ASP.NET Core

09 Dec, 2021

This post is my annual contribution to the 2021 C# Advent Calendar. Please check out all the great posts from our wonderful community!

Many systems require sending emails to notify users. And testing these notifications manually is a pain. So it’s one of the best use cases for integration testing. First, let’s create strongly typed model for our Welcome email:

1public class Welcome
2{
3    public string FullName { get; set; }
4}

And we create a Razor template for the email in EmailTemplates/Welcome.cshtml:

1@model EmailSnapshotTesting.EmailTemplates.Welcome
2@{
3    Layout = "~/EmailTemplates/_Layout.cshtml";
4}
5
6<h1>Welcome @Model.FullName</h1>
7<p>Welcome to our wonderful service!</p>

And this is how the layout is going to look like in EmaiTemplates/_Layout.cshtml:

 1<!DOCTYPE html>
 2
 3<html>
 4<head>
 5    <meta name="viewport" content="width=device-width" />
 6</head>
 7<body>
 8    <div>
 9        @RenderBody()
10    </div>
11</body>
12</html>

And then we create a service to send emails:

 1public class MailerService : IMailerService
 2{
 3    private readonly IEmailRenderer _renderer;
 4    private readonly IMailPostman _postman;
 5
 6    public MailerService(
 7        IEmailRenderer renderer,
 8        IMailPostman postman)
 9    {
10        _renderer = renderer;
11        _postman = postman;
12    }
13
14    public async Task SendWelcomeEmail(string address, Welcome welcome)
15    {
16        await SendEmail($"Welcome {welcome.FullName}!", address, welcome);
17    }
18
19    public async Task SendEmail<T>(string subject, string address, T model)
20    {
21        var html = await _renderer.Render(model);
22
23        await _postman.SendEmail(new Message
24        {
25            Subject = subject,
26            Address = address,
27            HtmlBody = html
28        });
29    }
30}

The MailerService needs an IEmailRenderer to get HTML content from the strongly typed model and an IMailPostman to send the emails.

Here is an implementation of IEmailRenderer that renders the Razor template we specified above:

 1using Microsoft.AspNetCore.Mvc;
 2using Microsoft.AspNetCore.Mvc.Abstractions;
 3using Microsoft.AspNetCore.Mvc.ModelBinding;
 4using Microsoft.AspNetCore.Mvc.Razor;
 5using Microsoft.AspNetCore.Mvc.Rendering;
 6using Microsoft.AspNetCore.Mvc.ViewFeatures;
 7
 8namespace EmailSnapshotTesting.Services;
 9
10// https://stackoverflow.com/a/49275145
11// https://ppolyzos.com/2016/09/09/asp-net-core-render-view-to-string/
12
13public class RazorEmailRenderer : IEmailRenderer
14{
15    private readonly IRazorViewEngine _razorViewEngine;
16    private readonly ITempDataProvider _tempDataProvider;
17    private readonly IServiceProvider _serviceProvider;
18
19    public RazorEmailRenderer(
20        IRazorViewEngine razorViewEngine,
21        ITempDataProvider tempDataProvider,
22        IServiceProvider serviceProvider)
23    {
24        _razorViewEngine = razorViewEngine;
25        _tempDataProvider = tempDataProvider;
26        _serviceProvider = serviceProvider;
27    }
28
29    public async Task<string> Render<T>(T model)
30    {
31        // Note: You can also support multiple languages by separating each locale into a folder
32        var viewPath = $"~/EmailTemplates/{typeof(T).Name}.cshtml";
33        var result = _razorViewEngine.GetView(null, viewPath, true);
34
35        if (result.Success != true)
36        {
37            var searchedLocations = string.Join("\n", result.SearchedLocations);
38            throw new InvalidOperationException($"Could not find this view: {viewPath}. Searched locations:\n{searchedLocations}");
39        }
40
41        var view = result.View;
42
43        var httpContext = new DefaultHttpContext();
44        httpContext.RequestServices = _serviceProvider;
45
46        var actionContext = new ActionContext(
47                httpContext,
48                httpContext.GetRouteData(),
49                new ActionDescriptor()
50            );
51
52        using (var writer = new StringWriter())
53        {
54            var viewDataDict = new ViewDataDictionary(
55                new EmptyModelMetadataProvider(),
56                new ModelStateDictionary());
57
58            viewDataDict.Model = model;
59
60            var viewContext = new ViewContext(
61                actionContext,
62                view,
63                viewDataDict,
64                new TempDataDictionary(
65                    httpContext.HttpContext,
66                    _tempDataProvider
67                ),
68                writer,
69                new HtmlHelperOptions { }
70            );
71
72            await view.RenderAsync(viewContext);
73
74            return writer.ToString();
75        }
76    }
77}

Now let’s create a fake implementation of the IEmailPostman for the integration tests:

 1public class FakePostman : IMailPostman
 2{
 3    public Task SendEmail(Message message)
 4    {
 5        LastMessage = message;
 6        return Task.CompletedTask;
 7    }
 8
 9    public Message LastMessage { get; set; }
10}

Let’s now register all of our services:

1builder.Services.AddScoped<IMailerService, MailerService>();
2builder.Services.AddScoped<IEmailRenderer, RazorEmailRenderer>();
3// In your project, you have to register a real postman in your app
4// and swap it our with this fake postman in the integration tests
5// by creating a custom WebApplicationFactory. For more info see: 
6// https://docs.microsoft.com/en-us/aspnet/core/test/integration-tests?view=aspnetcore-6.0#customize-webapplicationfactory
7builder.Services.AddScoped<IMailPostman, FakePostman>();

We create a test project called IntegrationTests using XUnit and inside the test project we create a folder called Snapshots to store the expected html results.

Then we can create our snapshot tests:

 1public class EmailTests : IClassFixture<WebApplicationFactory<Program>>
 2{
 3    private readonly IEmailRenderer _renderer;
 4    private readonly string _folderPath;
 5
 6    public EmailTests(WebApplicationFactory<Program> factory)
 7    {
 8        // Get the path for the snapshots folder
 9        var environment = factory.Services.GetRequiredService<IWebHostEnvironment>();
10        _folderPath = Path.Combine(environment.ContentRootPath, "../IntegrationTests/Snapshots");
11
12        var scope = factory.Services.CreateScope();
13        _renderer = scope.ServiceProvider.GetRequiredService<IEmailRenderer>();
14    }
15
16    [Fact]
17    public async Task CanSendWelcomeEmail()
18    {
19        var postman = new FakePostman();
20
21        var mailService = new MailerService(_renderer, postman);
22
23        await mailService.SendWelcomeEmail("person@example.com", new Welcome
24        {
25            FullName = "Example Person"
26        });
27
28        Assert.Equal("person@example.com", postman.LastMessage.Address);
29        Assert.Equal("Welcome Example Person!", postman.LastMessage.Subject);
30
31        await SaveToFile("Welcome.actual.html", postman.LastMessage.HtmlBody);
32        var expectedBody = await File.ReadAllTextAsync(Path.Combine(_folderPath, "Welcome.expected.html"));
33
34        Assert.Equal(Sanitize(postman.LastMessage.HtmlBody), Sanitize(expectedBody));
35    }
36
37    private string Sanitize(string text)
38    {
39        return text
40            .Replace("\r\n", "\n")
41            .Replace('\r', '\n');
42    }
43
44    private async Task SaveToFile(string name, string content)
45    {
46        var fullPath = Path.Combine(_folderPath, name);
47        Directory.CreateDirectory(Path.GetDirectoryName(fullPath));
48        await File.WriteAllTextAsync(fullPath, content);
49    }
50}

The first time your run CanSendWelcomeEmail it’s going to fail because IntegrationTests/Snapshots/Welcome.expected.html doesn’t exist. But it has created IntegrationTests/Snapshots/Welcome.actual.html. So go ahead and take a look at it, it should be something like this:

 1<!DOCTYPE html>
 2
 3<html>
 4<head>
 5    <meta name="viewport" content="width=device-width" />
 6</head>
 7<body>
 8    <div>
 9        <h1>Welcome Example Person</h1>
10        <p>Welcome to our wonderful service!</p>
11    </div>
12</body>
13</html>

You can test out the html using something like PutsMail or Testi@. If you like the result, rename it the file to IntegrationTests/Snapshots/Welcome.expected.html.

Because we don’t want git to track the actual results, you’ll have to add this line to your .gitignore file:

*.actual.html

Now you have snapshot tests for your email templates, whenever you change them, you can easily see the results without having to manually click through the UI to send the emails. This will make your feedback loop much faster.
